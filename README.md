# Pore-DCF: Pore Dynamic Collision Fingerprint

Pore-DCF is a Python descriptor for 2D segmented porous media. It simulates circular probe particles moving inside the pore space and extracts multiscale descriptors based on pore morphology, accessible regions, free paths, wall-collision statistics, angular entropy, and Fourier angular modes.

The repository contains two main scripts:

- `descriptor.py`: computes the numerical Pore-DCF descriptor and saves it as a CSV file.
- `interpretability.py`: generates interpretability plots and a textual summary using the segmented pore image and the CSV descriptor produced by `descriptor.py`.

## Input image format

The scripts expect a segmented RGB image where:

- green pixels represent pores;
- black pixels represent the solid matrix.

The default green-pixel detection criterion is implemented inside each script. If your segmentation uses a different shade of green, adjust the green-threshold parameters in the code.

## Repository structure

A simple recommended structure is:

```text
pore-dcf/
├── README.md
├── descriptor.py
├── interpretability.py
├── requirements.txt
├── .gitignore
├── data/
│   └── example_mask.png
└── outputs/
```

The `data/` and `outputs/` folders are optional. You may use any folder names, but the paths inside the scripts must match your local structure.

## Installation

Clone the repository and install the dependencies:

```bash
git clone https://github.com/YOUR_USERNAME/pore-dcf.git
cd pore-dcf
pip install -r requirements.txt
```

A minimal environment requires:

```text
numpy
pandas
pillow
scipy
scikit-image
matplotlib
```

## Step 1: Configure `descriptor.py`

Before running the descriptor, open `descriptor.py` and edit the input and output paths at the top of the file:

```python
IMAGE_PATH = "../poros/poros/patch_y7600_x53200_c0_mask.png"
OUTPUT_CSV = "pore_dcf_descriptor.csv"
```

Replace `IMAGE_PATH` with the path to your segmented pore image. Replace `OUTPUT_CSV` if you want to save the descriptor with another name or in another folder.

Also check the physical scale:

```python
PIXEL_SIZE_UM = 0.44
```

This value must correspond to the pixel size of your image in micrometers per pixel.

## Main descriptor parameters

The main parameters are defined at the top of `descriptor.py`:

```python
RADIUS_PERCENTILES = (1, 5, 10, 25, 50, 75, 90, 95)
NLAUNCHES = 1000
NSTEPS = 6000
FIXED_STEP_PX = None
STEP_FACTOR = 0.25
ANGULAR_NBINS = 180
MAX_FFT_MODE = 12
```

Meaning of the main parameters:

- `RADIUS_PERCENTILES`: percentiles of the internal distance map used to select the probe-particle radii.
- `NLAUNCHES`: number of probe particles launched for each selected radius.
- `NSTEPS`: number of simulation steps for each probe particle.
- `FIXED_STEP_PX`: fixed step size in pixels. If set to `None`, the step is computed automatically.
- `STEP_FACTOR`: factor used to compute the automatic step size when `FIXED_STEP_PX = None`.
- `ANGULAR_NBINS`: number of bins used for the collision-angle histogram.
- `MAX_FFT_MODE`: maximum Fourier angular mode recorded from the collision-angle distribution.

Higher values of `NLAUNCHES` and `NSTEPS` increase statistical robustness, but also increase runtime.

## Step 2: Run `descriptor.py`

After editing the paths and parameters, run:

```bash
python descriptor.py
```

The script will generate a CSV file, by default:

```text
pore_dcf_descriptor.csv
```

This CSV contains the full Pore-DCF descriptor for the input image.

## Step 3: Configure `interpretability.py`

The interpretability script requires two inputs:

1. the same segmented pore image used in `descriptor.py`;
2. the descriptor CSV produced by `descriptor.py`.

Open `interpretability.py` and edit:

```python
IMAGE_PATH = "../poros/poros/patch_y7600_x53200_c0_mask.png"
CSV_PATH = "pore_dcf_descriptor.csv"
OUTPUT_DIR = "pore_dcf_interpretability_outputs"
```

Use the same image path in `IMAGE_PATH`. Set `CSV_PATH` to the CSV generated by `descriptor.py`. Set `OUTPUT_DIR` to the folder where the figures and summary should be saved.

## Main interpretability parameters

The main parameters in `interpretability.py` are:

```python
RADIUS_PREFIXES = ["r1", "r5", "r10", "r25", "r50", "r75", "r90", "r95"]
TRAJECTORIES_PER_RADIUS = 100
NSTEPS_VISUAL = 1500
FIXED_STEP_PX = None
TRAJECTORY_ALPHA = 0.22
TRAJECTORY_LINEWIDTH = 0.35
PLOT_COLLISION_POINTS = True
FFT_MODES = list(range(1, 13))
FIGURE_DPI = 300
```

Meaning of the main parameters:

- `RADIUS_PREFIXES`: radii to visualize. These prefixes must exist in the CSV generated by `descriptor.py`.
- `TRAJECTORIES_PER_RADIUS`: number of visual trajectories generated for each radius.
- `NSTEPS_VISUAL`: number of simulation steps used only for visualization.
- `FIXED_STEP_PX`: fixed visual trajectory step size. If `None`, the step is computed automatically.
- `TRAJECTORY_ALPHA`: transparency of trajectory lines.
- `TRAJECTORY_LINEWIDTH`: width of trajectory lines.
- `PLOT_COLLISION_POINTS`: whether to plot collision points.
- `FFT_MODES`: Fourier modes displayed in the interpretability plot.
- `FIGURE_DPI`: output figure resolution.

The interpretability parameters do not change the descriptor CSV. They only control the visualization and explanatory outputs.

## Step 4: Run `interpretability.py`

After configuring the paths, run:

```bash
python interpretability.py
```

The script will create an output folder, by default:

```text
pore_dcf_interpretability_outputs/
```

Typical outputs include:

- binary mask visualization;
- internal distance map;
- probe-radius distribution;
- accessible fraction by radius;
- mean free path by radius;
- angular entropy by radius;
- Fourier angular-mode plot;
- trajectory plots for each radius;
- combined trajectory plots;
- textual interpretability summary.

## Important notes

- The image path must be correctly set in both scripts.
- The CSV path in `interpretability.py` must point to the descriptor generated by `descriptor.py`.
- The physical pixel size must match the image metadata.
- The selected `RADIUS_PREFIXES` in `interpretability.py` must correspond to the radius percentiles used in `descriptor.py`.
- Large values of `NLAUNCHES`, `NSTEPS`, `TRAJECTORIES_PER_RADIUS`, and `NSTEPS_VISUAL` may significantly increase runtime.
